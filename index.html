Diferencias entre Cloud Firestore y Realtime Database

Antes que nada, vamos a dar un breve concepto acerca de estas dos bases de datos NoSQL que ofrece Firebase. Primero, Cloud Firestore es la base de datos más reciente de Firebase para el desarrollo de apps para dispositivos móviles. Aprovecha lo mejor de Realtime Database con un modelo de datos nuevo y más intuitivo. Con respecto a Realtime Database, es la base de datos original de Firebase. Es una solución eficiente de baja latencia para las apps para dispositivos móviles que necesitan estados sincronizados entre los clientes en tiempo real.

Google recomienda usar Cloud Firestore a la mayoría de los desarrolladores que quieren comenzar proyectos nuevos. Esto debido a que Cloud Firestore ofrece más funciones, rendimiento y escalabilidad en una infraestructura diseñada para admitir funciones más potentes en las próximas actualizaciones. Habrá nuevos tipos de consultas, reglas de seguridad más sólidas y mejoras en el rendimiento entre las funciones avanzadas planificadas para Cloud Firestore.

Puedes usar ambas bases de datos dentro del mismo proyecto o la misma app de Firebase. Ambas bases de datos NoSQL pueden almacenar los mismos tipos de datos y las bibliotecas cliente funcionan de manera similar. Ten en cuenta las diferencias descritas anteriormente si decides usar ambas bases de datos en la app.
